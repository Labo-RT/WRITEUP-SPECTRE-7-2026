Cours 1 — User-Agent

Un User-Agent est l’identifiant envoyé par ton navigateur ou ton outil HTTP pour indiquer au serveur quel type de client effectue la requête. C’est une simple ligne de texte dans l’en-tête d’une requête HTTP, utilisée pour adapter l’affichage, reconnaître un bot, ou filtrer certains accès.

Changer son User-Agent revient simplement à dire au serveur “je suis ce client-là”. Rien de plus que modifier une chaîne dans les en-têtes HTTP.

Comment changer son User-Agent ?

  • curl
  • Burp Suite
  • Navigateur

CURL est un outil en ligne de commande pour envoyer des requêtes HTTP sans navigateur. Il permet de tester facilement un site et de modifier n’importe quel en-tête, dont le User-Agent.

Burp Suite est un proxy d’interception utilisé en cybersécurité. Il permet de voir et modifier chaque requête envoyée par ton navigateur, dont le User-Agent, avant de l’envoyer au serveur.

Navigateur : certains navigateurs permettent de modifier manuellement le User-Agent via les outils développeur ou une extension.

Exemples curl :

# Envoyer un User-Agent personnalisé
curl -A "SPECTRE-BOT-ALPHA" http://localhost/page-secrete

Cours 2 — robots.txt

Le fichier /robots.txt est un petit fichier texte placé à la racine d’un site web pour indiquer aux robots (comme les moteurs de recherche) ce qu’ils peuvent explorer ou non. C’est une simple recommandation : les robots respectueux le suivent, les autres l’ignorent.

Il fonctionne en définissant un User-Agent ciblé, suivi de règles comme Allow ou Disallow, qui indiquent quelles pages explorer ou éviter.

Exemple :

User-agent: *
Disallow: /admin/
